# Decision thresholds (GO / TUNE / ROLLBACK)

`tools/run_eval_cycle.py decision-eval` turns evaluation artifacts into an explicit rollout decision.
Run `tools/run_eval_cycle.py artifact-integrity` first to fail fast on missing/corrupt session artifacts before synthesis.

## Inputs

- `report.json` (primary source for PnL, fee drag, risk-event pressure)
- `drift-matrix.json` (primary source for drift severity)
- `conformal-summary.json` (optional model reliability signal)
- `tools/examples/decision-thresholds.json` (threshold schema)

If one or more artifacts are missing, the evaluator **does not fail**. It emits warnings and marks the missing dimension as `MISSING`.

## Threshold schema

Schema file: `tools/examples/decision-thresholds.json`

Policy metadata (for reproducibility):

- `policy.version` (recommended, required in strict validation mode)
- `policy.notes` (optional)
- `policy.changelog` (optional string/array)
- `policy.fingerprint` (canonical SHA-256 stamp; generated by helper command)

Required dimensions:

- `pnl_return_pct` (`go_min`, `tune_min`)
- `fee_drag_pct` (`go_max`, `tune_max`)
- `drift_severity` (`go_max`, `tune_max`, plus optional `normalizers` and `weights`)
- `risk_events` (`go_max`, `tune_max`, plus optional `weights`)

Optional:

- `conformal_miscoverage_pct` (`go_max`, `tune_max`)

## Decision policy

- Any `ROLLBACK` dimension => final `ROLLBACK`
- Else any `TUNE` dimension => final `TUNE`
- Else all present dimensions `GO` => final `GO`
- If all dimensions are missing => final `TUNE` with warning

`decision.json` now includes:

- `policy_version`
- `policy_fingerprint`

## Usage

```bash
# one-command macro (artifact-integrity runs before decision-eval)
python tools/run_eval_cycle.py --run-id 2026-04-19-paper-a full-cycle \
  --baseline-json artifacts/backtest-baseline.json \
  --require-conformal-summary \
  --strict

# individual commands
python tools/run_eval_cycle.py --run-id 2026-04-19-paper-a artifact-integrity

python tools/run_eval_cycle.py --run-id 2026-04-19-paper-a decision-eval
```

Generate rollback decision context packet (automation-safe + operator review):

```bash
python tools/run_eval_cycle.py --run-id 2026-04-19-paper-a rollback-packet \
  --decision-file logs/eval-cycle/2026-04-19-paper-a/decision.json \
  --integrity-file logs/eval-cycle/2026-04-19-paper-a/artifact-integrity.json \
  --confidence-file logs/eval-cycle/2026-04-19-paper-a/recommendation-confidence-v2.json \
  --out-file logs/eval-cycle/2026-04-19-paper-a/rollback-packet.json
```

Optional full-cycle hook:

```bash
python tools/run_eval_cycle.py --run-id 2026-04-19-paper-a full-cycle \
  --baseline-json artifacts/backtest-baseline.json \
  --generate-rollback-packet
```

Policy validation and stamping:

```bash
# Validate schema + metadata (allows legacy fallback)
python tools/run_eval_cycle.py threshold-policy-validate \
  --thresholds-json tools/examples/decision-thresholds.json

# Require explicit version + matching fingerprint
python tools/run_eval_cycle.py threshold-policy-validate \
  --thresholds-json tools/examples/decision-thresholds.json \
  --strict-policy-version --require-stamped

# Stamp canonical fingerprint into policy file (in-place)
python tools/run_eval_cycle.py threshold-policy-stamp \
  --thresholds-json tools/examples/decision-thresholds.json
```

With explicit files:

```bash
python tools/run_eval_cycle.py --run-id sample-run decision-eval \
  --thresholds-json tools/examples/decision-thresholds.json \
  --report-file tools/examples/decision-report.sample.json \
  --drift-file tools/examples/decision-drift.sample.json \
  --conformal-file tools/examples/decision-conformal.sample.json \
  --out-file logs/eval-cycle/sample-run/decision.sample.json
```

## Migration guidance

For existing threshold files that only define `schema_version` + `dimensions`:

1. Add a `policy` object with at least `version` (e.g. date/release tag).
2. Run `threshold-policy-stamp` to write `policy.fingerprint`.
3. Run `threshold-policy-validate --strict-policy-version --require-stamped`.
4. Re-run `decision-eval` so emitted decision artifacts include `policy_version` + `policy_fingerprint`.
